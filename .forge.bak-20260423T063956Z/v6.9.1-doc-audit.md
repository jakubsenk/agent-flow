### docs/guides/troubleshooting.md

- [BLOCKING] **No coverage of NEEDS_CLARIFICATION pause state** anywhere in the file. This is a major v6.9.0 user-facing behavior: when fixer or triage-analyst cannot proceed, the pipeline transitions to `paused` with a clarification question. Users who encounter a paused pipeline have no troubleshooting guidance. The block comment format differs from a regular `🔴 Pipeline Block` — it uses `pipeline-paused` state and requires `resume-ticket --clarification`. Add a new subsection under "Pipeline Issues":

  ```
  ### Pipeline Paused — Awaiting Clarification

  **Symptom:** The pipeline stops with a [ceos-agents] comment containing a question
  (no 🔴 block emoji — the issue is NOT in Blocked state, it is in the paused state).

  **Cause:** The fixer or triage-analyst encountered genuine ambiguity that cannot be resolved
  from the codebase or issue description. The agent emitted a NEEDS_CLARIFICATION signal.

  **Solution:**
  1. Read the clarification question in the issue comment
  2. Answer it with: /ceos-agents:resume-ticket ISSUE-ID --clarification "your answer"
  3. The pipeline resumes from where it paused (no re-analysis)

  Note: Autopilot automatically skips paused issues (they appear in the summary as "skipped:
  awaiting_clarification"). If the Pause timeout elapses (default 30 days), autopilot transitions
  the issue to aborted_by_system.
  ```

- [HIGH] **No coverage of autopilot circuit-breaker-open** log pattern. Users who see repeated `[WARN] Circuit breaker open` lines in their logs have no troubleshooting guidance. Add a troubleshooting entry:

  ```
  ### Autopilot: "[WARN] Circuit breaker open" in logs

  **Cause:** 3 consecutive webhook delivery failures (timeout, DNS, 4xx/5xx) triggered the
  in-memory circuit breaker. Remaining webhook calls for that pipeline run were suppressed.

  **Solution:** Check the Webhook URL in Automation Config. If it points to a dead or unreachable
  endpoint, either fix the endpoint or remove the Webhook URL from Notifications config.
  ```
