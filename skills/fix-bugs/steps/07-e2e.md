# Step 07 — E2E test-engineer (config-gated)

Dispatch `test-engineer --e2e` to run end-to-end test suite. This step is CONDITIONAL —
gated by the `E2E Test` config section and by a deployment health check.

## Skip conditions

Skip this entire step if ANY of the following is true:
- The `E2E Test` section is absent from Automation Config AND `test-engineer-e2e` is NOT in the
  profile's `Extra stages`.
- Stage `test-engineer-e2e` is in the profile's Skip stages.

When skipping: log `[SKIP] test-engineer-e2e ({reason})`, set `e2e_test.status = "skipped"` in
state.json (NEVER leave at `"pending"`), continue to step 08.

## Deployment guard (pre-E2E)

If `local_deployment_configured = true` AND the `E2E Test` section exists (or `Extra stages`
contains `test-engineer-e2e`):

**Pre-dispatch state write (deployment-verifier):**
- `deployment.started_at`      = current ISO-8601 UTC timestamp
- `deployment.model`           = `"sonnet"`
- `deployment.status`          = `"in_progress"`
- `deployment.agent_name`      = `"agent-flow:deployment-verifier"`
- `deployment.stage_name`      = `"deployment"`
- `deployment.dispatched_at`   = current ISO-8601 UTC timestamp
- `deployment.dispatch_witness` = sha256("agent-flow:deployment-verifier|sonnet|<prompt_head_128>")
- `deployment.tokens_used` = 0, `deployment.duration_ms` = 0, `deployment.tool_uses` = 0

Follow atomic write protocol from `../../../core/state-manager.md`.

### Agent Override injection

Before dispatch, check Agent Overrides: follow `../../../core/agent-override-injector.md`.
If `{Agent Overrides path}/deployment-verifier.toml` exists, append its rendered Markdown content
to the agent's context as `## Project-Specific Instructions`.

### Dispatch

You MUST invoke `Task(subagent_type='agent-flow:deployment-verifier', model='sonnet')`.
DO NOT inline-execute. Inline execution is a CONTRACT VIOLATION.

Inject Tier-1 variables: `EXPECTED_AGENT_NAME = "agent-flow:deployment-verifier"`,
`EXPECTED_STAGE_NAME = "deployment"`.

Context: `Action: start. Local Deployment config: Type = {Type}, Start command = {Start command},
Stop command = {Stop command}, Health check URL = {Health check URL},
Health check timeout = {Health check timeout}, Ports = {Ports}. Run directory: .agent-flow/{ISSUE-ID}/`

After dispatch returns, write `deployment.completed_at`, `deployment.tokens_used`,
`deployment.duration_ms`, `deployment.tool_uses`, `deployment.verdict`, `deployment.type`,
`deployment.result_path`. Follow atomic write protocol.

Verdict handling:
- `HEALTHY` or `SKIPPED` → continue to E2E dispatch below.
- `UNHEALTHY`, `PORT_CONFLICT`, `START_FAILED` → Block handler (step X).

## E2E test-engineer dispatch

Before dispatching, atomically write per-stage pre-dispatch fields:

- `e2e_test.started_at`      = current ISO-8601 UTC timestamp
- `e2e_test.model`           = `"sonnet"` (from `agents/test-engineer.md` frontmatter)
- `e2e_test.status`          = `"in_progress"`
- `e2e_test.agent_name`      = `"agent-flow:test-engineer"`
- `e2e_test.stage_name`      = `"e2e_test"`
- `e2e_test.dispatched_at`   = current ISO-8601 UTC timestamp
- `e2e_test.dispatch_witness` = sha256("agent-flow:test-engineer|sonnet|<prompt_head_128>")
- `e2e_test.tokens_used` = 0, `e2e_test.duration_ms` = 0, `e2e_test.tool_uses` = 0

Follow atomic write protocol.

### Agent Override injection

If `{Agent Overrides path}/test-engineer.toml` exists, append per `../../../core/agent-override-injector.md`.

### Dispatch

You MUST invoke `Task(subagent_type='agent-flow:test-engineer', model='sonnet')` with `--e2e` flag.
DO NOT inline-execute. Inline execution is a CONTRACT VIOLATION.

Inject Tier-1 variables: `EXPECTED_AGENT_NAME = "agent-flow:test-engineer"`,
`EXPECTED_STAGE_NAME = "e2e_test"`.

Context:
```
--e2e.
EXPECTED_AGENT_NAME = agent-flow:test-engineer
EXPECTED_STAGE_NAME = e2e_test
E2E framework = {Framework from E2E Test config}.
E2E command = {Command from E2E Test config}.
```

Failure → Block handler (step X).

### Post-dispatch state write

After dispatch returns:
- `e2e_test.completed_at` = current ISO-8601 UTC timestamp
- `e2e_test.tokens_used`  = `result.usage.total_tokens` (or 0 if absent)
- `e2e_test.duration_ms`  = completed_at epoch ms − started_at epoch ms
- `e2e_test.tool_uses`    = `result.usage.tool_uses` (or 0 if absent)
- `e2e_test.status`       = `"completed"`

Follow atomic write protocol.

## Step-completed webhook

On successful state commit, if Notifications → Webhook URL exists and `step-completed` is in On events:
fire `step-completed` with `step_name: "e2e_test"`, `iteration_count: 1`. Skip silently if stage was
skipped. On failure: log `[WARN] Webhook delivery failed: {error}`, continue.
