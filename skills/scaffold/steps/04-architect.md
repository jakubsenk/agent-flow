# Step 04: Architecture + Decomposition + Feature Plan Checkpoint

Dispatches the architect agent to decompose all spec epics into batched subtasks,
runs AC coverage check, then presents the Feature Plan Checkpoint.

## 04a. Epic Formatting

Read `spec/epics/*.md` — sort by filename (NN prefix ensures order).

For each epic, format user stories into the structured specification format that architect expects:
```
## Feature Specification
- **Summary:** {epic title}
- **Type:** epic ({N} sub-features)
- **Area:** {derived from epic content}
- **Acceptance Criteria:**
  1. {from user stories}
  2. {from user stories}
- **Scope:**
  - IN: {from epic description}
  - OUT: (none — all stories in this epic are in scope)
- **Dependencies:** {from epic Dependencies field}
```

## 04b. Architect Dispatch

**Pre-dispatch (COST-R4):** Read `model:` from `agents/architect.md` frontmatter (value: `opus`). Write to state.json atomically: `architect.started_at`, `architect.model = "opus"`, `architect.status = "in_progress"`, counters `0`.

Check Agent Overrides for `architect.md`.

You MUST invoke Task(subagent_type='ceos-agents:architect', model='opus'). DO NOT inline-execute.
Context: all formatted epic specifications + access to scaffolded codebase.
Explicit instruction:
  "Respect existing project structure generated by scaffolder.
   Decompose ALL epics into subtasks. Group subtasks into dependency-aware batches.
   Batch grouping: features with no unmet dependencies in the same batch.
   Batch size: 2–3 features per batch."

**Post-dispatch (COST-R2, COST-R3):** Defensive-read `result.usage`. Write `architect.completed_at`, `architect.tokens_used`, `architect.duration_ms`, `architect.tool_uses` (fallback `0`). Set `architect.status = "completed"`.

## 04c. Decomposition Validation

Read Decomposition config from generated CLAUDE.md:
- Max subtasks (default scaffold: 5 — lower than implement-feature's 7 because spec-writer pre-decomposes)
- Fail strategy (default: fail-fast)
- Commit strategy (default scaffold: individual)

Validate architect output:
- Total subtasks ≤ Max subtasks
- Dependencies form a DAG (no cycles)
- Each subtask has: id, title, scope, files, estimated_lines, depends_on, acceptance_criteria, `maps_to`

If validation fails → Block (follow `../../../core/block-handler.md`).

Update `state.json`: set `decomposition.status = "completed"`, write `decomposition.subtasks`, `decomposition.strategy`. Atomic write.

Note: `step-completed` webhook for `architect` is held until Feature Plan Checkpoint is approved.

## 04d. AC Coverage Check (per epic)

For each epic individually (using format `AC-{N}: {text}` per architect constraints):
1. Collect parent AC indices (1..N) from formatted epic specification
2. Collect all `maps_to` references from subtasks belonging to this epic
3. Compute set difference: parent AC indices not covered by any subtask's `maps_to`
4. If any parent AC is unmapped:
   - If MODE = yolo → Block ("Incomplete decomposition for epic {name} — unmapped AC")
   - Otherwise → display warning, ask: "Continue anyway? [Y/n]"

## 04e. Feature Plan Checkpoint

If MODE = yolo → skip checkpoint; fire held architect webhook immediately.

Display batch plan:
```
## Implementation Plan

### Batch 1: {name}
| # | Subtask | Files | ~Lines | Epic |
|---|---------|-------|--------|------|
| 1 | ...     | ...   | ~N     | 01-auth |

Total: {N} subtasks, {M} batches
Strategy: {fail_strategy}, commits: {commit_strategy}

Approve to start implementation? [Approve / Remove features / Abort]
```

If user aborts → STOP.
If user removes features → update task tree, re-display.

**Fire held `step-completed` webhook for `architect`:**
```bash
curl --proto "=http,https" --max-time 5 --retry 0 -X POST -H "Content-Type: application/json" \
  --data-binary @- "${Webhook_URL}" <<EOF
{"event":"step-completed","run_id":"${run_id}","issue_id":"${run_id}","step_name":"architect",
 "duration":${architect_duration_seconds},"iteration_count":1,"timestamp":"${ISO8601_UTC}"}
EOF
```
On failure: log `[WARN] Webhook delivery failed`, continue. Suppress if architect state.json write failed.
