## Věci k opravě

### ❌ Kritické (1)

**Soubor:** `commands/implement-feature.md`
**Problém:** Step 6g říká `"For features, the acceptance gate always runs (no conditional — features always have AC)."` — ale single-pass flow obsahuje `"Execute steps 6a–6e once"`, takže 6g se v single-pass vůbec nespustí. Slovo "always" je zavádějící.

**Oprava:** Upřesnit formulaci v 6g:
```
"For features within the decomposition loop, the acceptance gate always runs
(no threshold condition — unlike bugs, which require ≥3 AC or complexity ≥ M).
In single-pass mode, this step is skipped."
```

### ⚠️ Minor (1)

**Soubory:** `commands/fix-ticket.md` a `commands/fix-bugs.md`
**Problém:** AC coverage check sekce nezmiňuje explicitně formát `AC-{N}: {text}` pro maps_to matching. Čtenář musí přejít do implement-feature aby zjistil algoritmus.

**Oprava:** Přidat ke coverage check kroku poznámku:
```
Note: maps_to format is `AC-{N}: {text}` — matching is by index N only.
See AC matching algorithm in implement-feature step 5 for details.
```
