Scaffold commands nemají → inline handling stačí.

## 5. Decomposition Logic (Tasks 9-11) vs. Design Doc

### Konzistentní

- Decision criteria (thresholds) přesně odpovídají: `risk == HIGH`, `affected_files >= 4`, `diff > 60 AND files >= 3`, `independent_changes >= 2`
- Kombinovaný signál (`> 60 lines AND >= 3 files`) odpovídá `[ROZHODNUTO]` rozhodnutí v design docu
- DAG validace (cycle detection, topological sort, max_subtasks)
- Per-subtask rollback s restore_point
- Task tree YAML persistence v `.claude/decomposition/{ISSUE-ID}.yaml`
- Fail-fast strategie
- Interaktivní schválení před exekucí
- Context propagation mezi subtasky
- Integration test + squash commit po všech subtascích
- DECOMPOSE_PARTIAL checkpoint v resume-ticket

### Chybějící (ZAPRACOVÁNO)

Paralelní execution model (worktrees) — design doc Section 4 + Section 8 item 20. Přidán jako deferred Task 18.
