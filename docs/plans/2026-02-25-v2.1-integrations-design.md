## Přehled

Čtyři integrační funkce posouvající plugin od izolovaného nástroje k součásti širšího vývojového ekosystému:

1. **Webhook notifikace** — Slack/Teams/libovolný webhook upozorní tým na výsledek pipeline
2. **`/version-check` command** — porovná nainstalovanou verzi s nejnovější dostupnou
3. **Migration guide + semver policy** — pravidla co je major/minor/patch pro markdown plugin
4. **Odhad tokenů a nákladů** — hrubý odhad spotřeby po skončení pipeline

---

## Navrhovaný přístup

### 1. Webhook notifikace

**Co a kdy notifikovat:**

| Událost | Kdy | Obsah zprávy |
|---------|-----|--------------|
| Pipeline dokončena | Po publisher / po summary | Issue ID, PR link, počet fixnutých/blocknutých |
| Issue zablokováno | Kdykoli agent nastaví Blocked | Issue ID, agent, důvod |
| PR vytvořeno | Po publisher | Issue ID, PR link, branch |

**Konfigurace** — nová volitelná sekce v `## Automation Config`:

```markdown
### Notifications (optional)
| Klíč | Hodnota |
|------|---------|
| Webhook URL | `https://hooks.slack.com/services/...` |
| On events | `pipeline-complete, issue-blocked, pr-created` |
```

**Payload** — generický JSON:

```json
{
  "event": "pipeline-complete | issue-blocked | pr-created",
  "issue_id": "PROJECT-123",
  "status": "success | blocked",
  "pr_url": "https://...",
  "agent": "reviewer",
  "reason": "...",
  "timestamp": "2026-02-25T14:30:00Z"
}
```

**Implementace:** `Bash` `curl --max-time 5 --retry 0`. Selhání = varování, nesmí zastavit pipeline.

### 2. `/version-check` command

**Flow:**
1. Přečti nainstalovanou verzi z `plugin.json`
2. `git ls-remote <repository> refs/tags/v*` → seřaď semver sestupně → vezmi první
3. Porovnej a zobraz výsledek (aktuální / dostupná aktualizace / lokálně novější)
4. Při selhání sítě zobraz jen lokální verzi s varováním

**Allowed tools:** `Read, Bash`

**Nový soubor:** `commands/version-check.md`

---

### 3. Migration guide + semver policy

**Semver policy pro markdown plugin:**

| Úroveň | Trigger | Příklady |
|--------|---------|---------|
| MAJOR (X.0.0) | Breaking change v Automation Config kontraktu — přidání povinného klíče, přejmenování sekce | Nový povinný klíč |
| MINOR (X.Y.0) | Nová funkce zpětně kompatibilní — nový volitelný klíč, nový command/agent | `/version-check` |
| PATCH (X.Y.Z) | Oprava chování bez změny kontraktu | Text agenta, logika commandu |

**Klíčové pravidlo:** Přidání povinného klíče do Automation Config = MAJOR. Přidání volitelné sekce = MINOR.

**Soubory:**
- `CHANGELOG.md` v kořeni (Keep a Changelog formát)
- `MIGRATION.md` vznikne až při prvním MAJOR release
- `CLAUDE.md` — nová sekce `## Versioning Policy`

### 4. Odhad spotřeby tokenů a nákladů

**Model odhadu (na jeden issue):**

| Agent | Model | Tokeny (hrubý odhad) |
|-------|-------|----------------------|
| triage-analyst | sonnet | ~8 000 |
| code-analyst | sonnet | ~15 000 |
| fixer | opus | ~30 000 |
| reviewer (avg 2× iterace) | opus | ~40 000 |
| test-engineer | sonnet | ~12 000 |
| e2e-test-engineer | sonnet | ~10 000 |
| publisher | haiku | ~4 000 |
| **Celkem** | | **~119 000 tokenů** |

**Zobrazení:** Post-run summary (po dokončení pipeline). Formát:
```
Odhadovaná spotřeba: ~595 000 tokenů (5 issues × ~119 000)
Odhadovaná cena: ~$2.50–$8.00 USD
(Odhad je orientační — skutečné náklady mohou být 2–5× vyšší/nižší)
```

## Alternativy

**Notifikace:**
- Rozšíření `publisher` agenta — ale publisher se nevolá při Blocked. Zamítnuto.
- Nový `notifier` agent — overkill pro v2.1. Zamítnuto.
- **Přímé `Bash curl` v commandu** — nejjednodušší. Přijato.

**`/version-check`:**
- `git ls-remote` (přijato): spolehlivé, bez HTTP autentizace.
- HTTP GET na `marketplace.json`: závislé na Gitea HTTP. Zamítnuto.

**Token odhad:**
- Pre-run estimate s potvrzením: friction při opakovaném použití. Zamítnuto.
- **Post-run summary** (přijato): informativní, bez interference.

## Designová rozhodnutí

**D1: Notifikace jsou opt-in.** Sekce `Notifications` je volitelná = MINOR change.

**D2: Webhook payload je generický JSON.** Slack-specifický formatting mimo scope.

**D3: Token odhad je post-run.** Pre-run by přerušoval workflow.

**D4: `MIGRATION.md` vznikne při prvním MAJOR release.** Prázdný dokument nemá smysl.

**D5: Semver policy v `CLAUDE.md`.** Authoring guide — přispívající vidí pravidla ihned.

**D6: `/version-check` funguje jen v CLAUDE-agents repozitáři.** Guard na existenci `plugin.json`.

## Vyřešené otázky (2026-02-25)

**O1: Git tagging:** ANO — přidat `git tag v{verze}` do `/version-bump`. `/version-check` na tom závisí.

**O2: Auth pro `git ls-remote`:** Vyžaduje SSH klíč nebo HTTPS token (stejný jako pro clone). Dokumentovat v `docs/setup/`.

**O3: Webhook selhání:** `curl --max-time 5`, jeden pokus, nikdy nezastaví pipeline.

**O4: Přesnost token odhadu:** Orientační s "2–5× variance" disclaimerem. Zpřesnit po reálných runech.

**O5: Ceny modelů:** Hardkódovat s datumem. `Model pricing` v Automation Config je YAGNI.

**O6: `/version-bump` git tagging:** ANO — rozhodnuto.
