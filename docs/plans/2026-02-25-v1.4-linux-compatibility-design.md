# Design: v1.4.0 — Linux kompatibilita

**Datum:** 2026-02-25
**Status:** APPROVED (2026-02-25)

---

## Přehled

Roadmap předpokládal opravu path separátorů (backslash → forward slash) a ověření Forgejo MCP serveru na Linuxu. Audit všech souborů v pluginu ale odhalil **klíčový nález**: žádné hardcoded Windows cesty, žádné backslash separátory, žádné platform-specific shell příkazy nikde v pluginu neexistují.

Plugin je čistý markdown — Claude Code's runtime (ne plugin) generuje OS-appropriate příkazy. Concern z roadmapy je **non-issue pro samotný plugin**.

---

## Audit stávajícího kódu

### Prohledané soubory

- Všechny soubory v `agents/` (7 agentů)
- Všechny soubory v `commands/` (6 commands)
- `skills/bug-workflow/SKILL.md`
- `README.md`, `CLAUDE.md`

### Nález

**Žádné platform-specific problémy nalezeny:**
- Žádné hardcoded cesty (`C:\...`, `/home/...`)
- Žádné backslash separátory
- Žádné platform-specific shell příkazy (vše je delegováno na Claude Code runtime)
- Žádné binární reference (MCP servery se konfigurují v `.mcp.json`, ne v pluginu)

---

## Reálná cross-platform práce

Skutečné platform-specific problémy jsou **mimo plugin** — v dokumentaci a konfiguraci:

### 1. Forgejo MCP server binary na Linuxu

Windows a Linux se liší v:
- Název binárky (`.exe` vs bez přípony)
- Cesta k binárce
- Permissions (`chmod +x` na Linuxu)
- `.mcp.json` cesta (`~/.claude/.mcp.json` vs `%USERPROFILE%\.claude\.mcp.json`)

→ Řešení: **dokumentace** v `docs/setup/` (viz v1.2.0 design)

### 2. Worktree base path v Automation Config

Projekty mohou mít v Automation Config absolutní cestu: `Worktree base path: C:\worktrees`. Na Linuxu toto nefunguje.

→ Řešení: dokumentovat, že cesta má být **relativní** (např. `.worktrees/`)

### 3. Pipeline testování na Linuxu

Neexistuje automatický test — ověření musí být manuální.

→ Řešení: manuální test checklist

---

## Navrhovaný přístup

### Doporučení: Sloučit v1.4.0 s v1.2.0

Oba jsou primárně **dokumentační** — v1.2.0 vytváří `docs/setup/` s instalačním návodem, v1.4.0 přidává Linux-specific kroky do téhož. Oddělená verze pro "opravu neexistujících path problémů" nedává smysl.

**Konkrétní kroky:**
1. V `docs/setup/installation.md` (z v1.2.0) přidat Linux sekce
2. V `docs/setup/mcp-configuration.md` (z v1.2.0) přidat Linux MCP binary setup
3. V README přidat poznámku o worktree path formátu (relativní cesty)
4. Manuální test checklist v `docs/setup/` nebo `docs/plans/`

## Alternativy

### A: Automatická detekce OS v command instrukcích

Command by obsahoval instrukci: "Zjisti OS a podle toho generuj příkazy." Zamítnuto: Claude Code runtime to již dělá automaticky. Plugin nepotřebuje OS detekci.

### B: Samostatná verze v1.4.0 s vlastním scopem

Zamítnuto: po auditu neexistuje žádný kód k opravě. Vše je dokumentace → sloučit s v1.2.0.

### C: Cross-platform test suite

Automatické testy na Windows i Linuxu. Zamítnuto pro v1.4.0: vyžaduje CI/CD infrastrukturu, mock MCP servery — to je spíše v1.7.0 (E2E test suite).

## Designová rozhodnutí

**1. Plugin je platform-agnostic.** Žádné opravy kódu nejsou potřeba. Vše řeší Claude Code runtime.

**2. Sloučení s v1.2.0.** Linux dokumentace je přirozenou součástí instalačních docs, ne separátní verze.

**3. Relativní worktree cesty.** Dokumentace bude doporučovat relativní cesty v Automation Config.

---

## Vyřešené otázky (2026-02-25)

**1. Forgejo MCP server:** `forgejo-mcp` — oficiální Go binary z [codeberg.org/forgejo/forgejo-mcp/releases](https://codeberg.org/forgejo/forgejo-mcp/releases). Na Linuxu: stáhnout linux-amd64 binary, `chmod +x`, uložit do `bin/forgejo-mcp`. Ověřeno na BIFITO (Windows verze v0.0.7).

**2. macOS:** Explicitně uvést jako "nepodporovaný, ale pravděpodobně fungující" (analogické Linuxu).

**3. Manuální test na Linuxu:** Po dokončení dokumentace, před release. Checklist v `docs/setup/cross-platform-checklist.md`.
