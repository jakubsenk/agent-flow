# Design: v1.8.0 — Pipeline Resume

**Datum:** 2026-02-25
**Status:** APPROVED (2026-02-25)

---

## Přehled

Pipeline má 9 kroků a může selhat kdekoli. Restart od nuly plýtvá penězi (opakované LLM calls), časem (5–15 minut) a může způsobit duplicitní komentáře v issue trackeru. Cílem je umožnit obnovení pipeline z místa selhání bez re-analýzy.

---

## Klíčové omezení

Plugin je **stateless** — čistý markdown, žádný runtime. Stav musí být odvozen výhradně z externích signálů:
- Issue tracker stav a komentáře
- Git branch existence a commit historie
- Otevřené PR

---

## Navrhovaný přístup

### Nový command: `/resume-ticket`

**Proč nový command, ne flag `--resume` na `/fix-ticket`:**
Commands nemají formální flag parsing mechanismus. Samostatný command je čistší a jednoznačnější.

**Allowed tools:** `mcp__*, Bash, Read, Write, Edit, Grep, Glob, Task`

### Jak funguje

1. Přečte issue tracker stav, komentáře a git branch stav
2. Určí checkpoint z 6-úrovňové tabulky
3. Přeskočí všechny kroky před checkpointem
4. Spustí pipeline od checkpointu dál

### Checkpoint tabulka

| Checkpoint | Signál | Přeskočí |
|-----------|--------|---------|
| `FRESH` | Žádná branch, žádné komentáře | Nic — běží celá pipeline |
| `POST_TRIAGE` | Existuje triage checkpoint komentář | Triage |
| `POST_ANALYSIS` | Existuje branch + triage komentář | Triage + code-analyst |
| `POST_FIX` | Branch s commity nad base branch | Triage + code-analyst + fixer |
| `POST_REVIEW` | Branch + reviewer approval komentář | Triage + code-analyst + fixer + reviewer |
| `PUBLISHED` | Existuje otevřené PR | Celá pipeline — jen zobrazí stav |

### Persistence mechanismus: Checkpoint komentáře

Jediná změna v agentech: `triage-analyst` přidá strukturovaný checkpoint komentář po úspěšném dokončení:

```
[CLAUDE-agents] Triage dokončen. Severity: {severity}. Area: {area}.
```

Tento komentář je "persistence vrstva" — `/resume-ticket` ho hledá, aby věděl, že triage proběhl.

Code-analyst nepotřebuje checkpoint — jeho výstup je inferován z existence branch.

### Detection logic

```
if PR exists for branch → PUBLISHED
else if branch has commits above base → POST_FIX (or POST_REVIEW if reviewer comment exists)
else if branch exists + triage comment → POST_ANALYSIS
else if triage comment exists → POST_TRIAGE
else → FRESH
```

## Alternativy

### A1: Lokální state file (`.claude-agents-state/`)

Persistentní soubor s posledním checkpointem.
**Zamítnuto:** Nepřežije mezi Claude Code sessions. Anti-pattern pro markdown plugin.

### A2: Flag `--resume` na `/fix-ticket`

**Zamítnuto:** Commands nemají formální flag parsing. Samostatný command je čistší.

### A3: Vždy re-run everything

**Zamítnuto:** Příliš drahé, duplicitní issue komentáře.

### A4: Idempotentní kroky v `/fix-ticket`

Každý krok zkontroluje, zda už proběhl, a přeskočí se.
**Zamítnuto:** Nezajistí úsporu LLM tokenů (agent musí být spuštěn, aby detekoval idempotenci). A file edity nelze garantovat idempotentní.

## Designová rozhodnutí

**1. Issue tracker komentář jako persistence.** Přežije sessions, nevyžaduje runtime. Komentář je strukturovaný prefix `[CLAUDE-agents]` pro snadnou detekci.

**2. Code-analyst re-runs na prázdné branch.** Pokud branch existuje ale nemá commity, code-analyst se spustí znovu — jeho výstup není persistentní mimo session.

**3. Publish zůstává manuální.** `/resume-ticket` nezavolá publisher automaticky — uživatel musí explicitně rozhodnout o publish.

**4. Detekce je best-effort.** Heuristika nemusí být 100% přesná. Horší případ: re-run jednoho kroku navíc — ne katastrofa.

---

## Vyřešené otázky (2026-02-25)

**1. Code-analyst output persistence:** Fixer si přečte kód sám. Code-analyst je optimalizace, ne hard dependency.

**2. Triage checkpoint formát:** `[CLAUDE-agents] Triage dokončen. Severity: {severity}. Area: {area}.` Konzistentní s Block Template prefixem `[CLAUDE-agents]`.

**3. Batch resume pro fix-bugs:** Mimo scope v1.8. Každý ticket se resumuje samostatně.

**4. Blocked issue handling:** Ano, zkusit znovu s varováním "issue was previously blocked by {agent}: {důvod}".

**5. Checkpoint comment versioning:** Prefix `[CLAUDE-agents]` je stabilní, obsah best-effort parsing.
