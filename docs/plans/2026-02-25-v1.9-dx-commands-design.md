# Design: v1.9.0 — DX commands

**Datum:** 2026-02-25
**Status:** APPROVED (2026-02-25)

---

## Přehled

v1.9.0 přidává tři funkce zaměřené na zlepšení developer experience (DX):

| Funkce | Popis |
|--------|-------|
| `/status` | Přehledová tabulka rozpracovaných issues — stav pipeline, branch, PR |
| `/onboard` | Interaktivní průvodce, který vygeneruje `## Automation Config` do CLAUDE.md |
| `/changelog` | Automatické generování changelogu z merged PR |

---

## Navrhovaný přístup

### 1. `/status` command

**Co dělá:** Dotáže se issue trackeru na issues v aktivních stavech (In Progress, Blocked, For Review) a zobrazí přehledovou tabulku s pipeline fází, branch a PR.

**Formát výstupu:**

| Issue | Název | Fáze | Branch | PR |
|-------|-------|------|--------|----|
| PROJ-42 | Login page crash on iOS | In Progress (fix) | PROJ-42-login-crash | #87 (open) |
| PROJ-51 | Export fails for >10k rows | In Progress (review) | PROJ-51-export-fix | — |
| PROJ-63 | Dark mode flicker | For Review | PROJ-63-dark-mode | #91 (open) |

**Jak zjistí pipeline fázi:** Kombinace issue tracker stavu a existence PR v source controlu.

| Issue tracker stav | PR existuje? | Zobrazená fáze |
|--------------------|--------------|----------------|
| In Progress | Ne | `in progress` |
| In Progress | Ano | `in progress (PR open)` |
| Blocked | Ne | `blocked` |
| For Review | Ano | `for review` |

**Kroky commandu:**
1. Čti Automation Config z CLAUDE.md
2. Dotáže se issue trackeru na issues v aktivních stavech
3. Pro každý issue: zjisti branch (dle branch naming pattern) a PR (dle source control MCP)
4. Zobraz tabulku + součty: `N in progress | M blocked | K for review`

**Allowed tools:** `mcp__*, Read, Grep`
**Soubor:** `commands/status.md`

---

### 2. `/onboard` command

**Co dělá:** Interaktivní průvodce, který se postupně zeptá na všechny parametry Automation Config a vygeneruje hotový blok připravený ke zkopírování do CLAUDE.md.

**Klíčové rozhodnutí: uživatel si zvolí výstup.** Po shromáždění všech odpovědí command nabídne dvě možnosti:
1. **Vypsat do chatu** (default) — uživatel zkopíruje sám
2. **Zapsat do CLAUDE.md** — command přidá `## Automation Config` blok na konec existujícího CLAUDE.md

Pokud uživatel zvolí zápis a `## Automation Config` v souboru již existuje, command varuje a nabídne přepis nebo zrušení.

**Otázky průvodce (v pořadí):**

| # | Sekce | Otázka | Default |
|---|-------|--------|---------|
| 1 | Issue Tracker | Jaký issue tracker používáš? | — |
| 2 | Issue Tracker | URL instance | — |
| 3 | Issue Tracker | Název projektu / project key | — |
| 4 | Issue Tracker | Query pro otevřené bugy | závisí na trackeru |
| 5-8 | Issue Tracker | Stavy: In Progress, Blocked, For Review, On start set | defaulty |
| 9 | Source Control | Hostname repozitáře | — |
| 10 | Source Control | Hlavní branch | `main` |
| 11 | Source Control | Vzor pro branch naming | `{issue}-fix` |
| 12-13 | PR Rules | Labely, šablona PR | — |
| 14-15 | Build & Test | Build command, Test command | — |
| 16-21 | Volitelné | E2E testy, Worktrees | opt-in |

**Allowed tools:** `Read, Glob, Write, Edit`
**Soubor:** `commands/onboard.md`

---

### 3. `/changelog` command

**Co dělá:** Vygeneruje changelog z merged PR od posledního git tagu. Zapíše do `CHANGELOG.md` (Keep a Changelog formát).

**Odkud bere data:**
- `git log {last_tag}..HEAD --oneline --merges`
- Source control MCP — metadata k PR
- `git tag --sort=-version:refname | head -1`

**Formát:**

```
## [1.2.0] — 2026-02-25

### Nové funkce
- feat: popis z PR titulu (#42)

### Opravy
- fix: popis z PR titulu (#39)

### Interní
- chore/docs/refactor: popis (#40)
```

**Kategorizace** dle Conventional Commits prefixů.

**Allowed tools:** `mcp__*, Bash, Read, Write, Edit, Glob`
**Soubor:** `commands/changelog.md`

## Alternativy

### `/status`
- **Agent místo commandu:** Zbytečná složitost pro read-only přehled. Zamítnuto.

### `/onboard`
- **Pouze automatická editace CLAUDE.md:** Destruktivní write bez záruky zachování struktury. Zamítnuto jako default — ale nabídnuto jako opt-in volba uživateli.
- **Agent s autodetekcí technologie:** Chytřejší, ale dražší. Odloženo na pozdější verzi.

### Changelog
- **Z commit messages místo PR:** Méně konzistentní. Zamítnuto jako primární přístup.
- **Pouze release notes (bez CHANGELOG.md):** Neverzovaný v repozitáři. Zamítnuto.

## Designová rozhodnutí

**D1: `/status` bez cachování — vždy live data.** Plugin je pure markdown bez persistence.

**D2: `/onboard` nedetekuje technologii projektu.** YAGNI — základní průvodce je cíleně jednoduchý.

**D3: Model pro `/onboard` a `/changelog` je haiku.** Mechanické — shromáždění vstupů + formátování výstupu.

**D4: `/status` nezobrazuje Done/Closed issues.** Přehled je pro aktivní práci.

**D5: Changelog formát Keep a Changelog.** Standardní, strukturovaný, dobře známý.

## Vyřešené otázky (2026-02-25)

**O1: Branch detekce v `/status`:** Hledat remote branches obsahující issue ID jako substring. Spolehlivé — branch naming pattern vždy obsahuje issue ID.

**O2: Validace v `/onboard`:** Ne. Validace patří do `/check-setup`. Onboard jen shromažďuje vstupy.

**O3: CHANGELOG.md vytvoření:** Ano, `/changelog` vytvoří soubor pokud neexistuje.

**O4: PR bez Conventional Commits:** Fallback sekce "Ostatní změny".

**O5: Issues bez branch v `/status`:** Ano, zobrazit s `—` v Branch a PR sloupcích.

**O6 (nové): `/onboard` výstup:** Uživatel si zvolí — vypsat do chatu (default) nebo zapsat přímo do CLAUDE.md. Při zápisu ověřit, že `## Automation Config` neexistuje (varování + nabídka přepisu). `allowed-tools` rozšířeny o `Write, Edit`.
