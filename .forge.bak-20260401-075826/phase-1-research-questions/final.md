# Phase 1: Research Questions — Synthesized Final

## Summary

31 research questions across 10 areas, generated by 2 research agents analyzing the full ceos-agents codebase. Questions are organized by priority: critical gaps first, then structural risks, then coverage improvements.

## Priority 1: Critical Gaps (bugs and contradictions found)

### RQ-01: 19th agent missing from test hardcoded lists
**Area:** 1, 7 | **Files:** tests/scenarios/frontmatter-completeness.sh, tests/scenarios/model-assignment.sh, tests/scenarios/section-order.sh
**Finding:** All three tests hardcode 18-agent lists. deployment-verifier (19th, added v5.3.0) is absent. Tests pass but don't validate the newest agent.

### RQ-02: core/mcp-detection.md excluded from core-include-refs.sh
**Area:** 5, 7 | **Files:** tests/scenarios/core-include-refs.sh, core/mcp-detection.md
**Finding:** CORE_FILES array lists 10 files. mcp-detection.md is the 11th core contract and has zero test coverage.

### RQ-03: Feature pipeline acceptance gate contradicts CLAUDE.md
**Area:** 10 | **Files:** commands/implement-feature.md (Step 6g), CLAUDE.md (Feature Pipeline)
**Finding:** CLAUDE.md says "Acceptance gate (always)" for features. implement-feature.md skips it in single-pass mode. Contradiction.

### RQ-04: Root cause iterations missing from config-reader.md
**Area:** 6 | **Files:** CLAUDE.md, core/config-reader.md
**Finding:** CLAUDE.md documents `Root cause iterations` (default: 3). config-reader.md omits it entirely. code-analyst always falls back to hardcoded default.

### RQ-05: State field reuse undocumented — triage.* reused for spec-analyst in feature pipeline
**Area:** 4, 10 | **Files:** commands/implement-feature.md, state/schema.md
**Finding:** implement-feature reuses triage.status for spec-analyst output. Undocumented in schema. resume-ticket could misinterpret.

## Priority 2: Structural Risks (inconsistencies between files)

### RQ-06: fix-bugs.md missing Config Validity Gate (Step 0b)
**Area:** 2 | **Files:** commands/fix-bugs.md, commands/fix-ticket.md, commands/implement-feature.md
**Finding:** fix-ticket and implement-feature have Step 0b. fix-bugs has none. Deliberate or missing?

### RQ-07: fixer-reviewer-loop.md reference placement inconsistent across pipelines
**Area:** 2 | **Files:** commands/fix-ticket.md, commands/fix-bugs.md, commands/implement-feature.md, core/fixer-reviewer-loop.md
**Finding:** Reference appears BEFORE reviewer step in fix-ticket but INSIDE reviewer step in fix-bugs and implement-feature.

### RQ-08: Block handler rollback scope inconsistently documented
**Area:** 5 | **Files:** core/block-handler.md, commands/fix-ticket.md, commands/fix-bugs.md, commands/implement-feature.md
**Finding:** fix-bugs explicitly re-documents the no-rollback-for-analysis-agents rule. fix-ticket and implement-feature just delegate to core. Risk of unintended rollback.

### RQ-09: Decomposition state writes missing from bug pipelines
**Area:** 4 | **Files:** commands/fix-ticket.md, commands/fix-bugs.md, state/schema.md
**Finding:** fix-ticket and fix-bugs have decomposition steps but never write decomposition.* to state.json. implement-feature and scaffold do.

### RQ-10: implement-feature references decomposition-heuristics.md which expects bug-pipeline inputs
**Area:** 7 | **Files:** commands/implement-feature.md, core/decomposition-heuristics.md
**Finding:** decomposition-heuristics Input Contract expects code_analyst_output (bug-pipeline only). Feature pipeline uses spec-analyst/architect.

### RQ-11: Deployment port validation duplicated between command and agent
**Area:** 9 | **Files:** commands/check-deploy.md, agents/deployment-verifier.md
**Finding:** Both implement "digits only, 1-65535" independently. No consistency test.

### RQ-12: Profile-parser accepts triage/code-analyst stages for feature pipeline where they don't exist
**Area:** 7 | **Files:** core/profile-parser.md, CLAUDE.md
**Finding:** Valid stage set includes triage and code-analyst regardless of pipeline type. Silent no-op in feature pipeline.

## Priority 3: Coverage Improvements (untested properties)

### RQ-13: State.json phase fields have no specific-field tests
**Area:** 1 | **Files:** tests/scenarios/state-schema.sh, state/schema.md
**Finding:** state-schema.sh only checks count-based references (≥5). No test validates specific field names like triage.severity, acceptance_gate.verdict, etc.

### RQ-14: Zero test coverage for Config Validity Gate blocking behavior
**Area:** 1 | **Files:** commands/fix-ticket.md (Step 0b), commands/implement-feature.md (Step 0b)
**Finding:** Step 0b blocks on TODO placeholders. No test validates this behavior.

### RQ-15: Zero test coverage for --yolo flag logic
**Area:** 1 | **Files:** commands/fix-ticket.md, commands/implement-feature.md
**Finding:** --yolo auto-approves and auto-publishes. No test validates this.

### RQ-16: Zero feature pipeline (implement-feature) dedicated tests
**Area:** 10 | **Files:** commands/implement-feature.md, tests/scenarios/
**Finding:** Only tested for core reference counts and Fix Verification. No test validates spec-analyst dispatch, architect, decomposition, AC propagation.

### RQ-17: Zero deployment-verifier/check-deploy test coverage
**Area:** 9 | **Files:** agents/deployment-verifier.md, commands/check-deploy.md
**Finding:** No test validates frontmatter, model, verdict set, state writes, or port validation.

### RQ-18: Mock project missing 9 of 16 optional config sections
**Area:** 8 | **Files:** tests/mock-project/CLAUDE.md, CLAUDE.md
**Finding:** Missing: Module Docs, Custom Agents, Notifications, E2E Test, Browser Verification, Error Handling, Extra labels, Agent Overrides, Local Deployment.

### RQ-19: Mock project Retry Limits missing Spec iterations and Root cause iterations
**Area:** 8 | **Files:** tests/mock-project/CLAUDE.md

### RQ-20: Verify command optional-within-required null handling untested
**Area:** 6 | **Files:** CLAUDE.md, core/config-reader.md, core/fix-verification.md
**Finding:** No test validates that missing Verify command is treated as null (not error).

### RQ-21: PR Description Template backtick fence handling unspecified
**Area:** 6 | **Files:** CLAUDE.md, core/config-reader.md, tests/mock-project/CLAUDE.md

### RQ-22: Feature Workflow fallback chain undocumented in config-reader
**Area:** 6 | **Files:** CLAUDE.md, core/config-reader.md, commands/implement-feature.md
**Finding:** implement-feature falls back from feature.on_start_set to issue_tracker.on_start_set. Not in config-reader contract.

### RQ-23: Rollback-agent namespace prefix inconsistent in implement-feature
**Area:** 3 | **Files:** commands/implement-feature.md, commands/fix-bugs.md

### RQ-25: acceptance_gate.status enum (skipped vs not_applicable) inconsistent
**Area:** 2 | **Files:** commands/fix-ticket.md, commands/fix-bugs.md, commands/implement-feature.md, state/schema.md
